A B+-tree is a multi-level index structure like a B-tree except that *all data is stored at the leaf nodes of the resulting tree* instead of (possibly) within the tree itself. Thus, a B+-tree has two distinct node types:

1.  **Interior nodes:** Store only search keys and pointers to other interior nodes or leaf nodes. They are simply a hierarchy of multi-level index used only to direct searches.
2.  **Leaf nodes:** Store keys and pointers to the data records (or the data records themselves). Each leaf node contains a pointer to the next leaf node which makes it easy to chain together and maintain the data records in "sequential" order for sequential processing.

### Advantages of B+-Trees

| Feature | Description |
| :--- | :--- |
| **Self-balancing** | B+-trees automatically create or destroy index levels as the data file changes. |
| **Dynamic Structure** | They are a dynamic structure; inserts and deletes leave the tree height-balanced, meaning every leaf has the same depth. |
| **Logarithmic Search** | The search time is the same for any search key and is logarithmic. |
| **Efficient Range Searches** | They support both equality searches and range-searches efficiently by chaining leaf nodes. |
