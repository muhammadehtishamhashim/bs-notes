### Static Hashing
In static hashing, when a search-key value is provided the hash function always computes the same address. The numbers of buckets provided remain the same at all times. 
*   **Insertion:** The hash function computes the bucket address for the search key where the record will be stored.
*   **Bucket Overflow:** The condition of bucket-overflow is known as a collision. This is a fatal state for any static hash function. To resolve this, the system uses **Overflow Chaining** (Closed Hashing), where a new bucket is allocated and linked after the previous one, or **Linear Probing** (Open Hashing), where the next free bucket is allocated.

### Dynamic Hashing
The problem with static hashing is that it does not expand or shrink dynamically as the size of the database grows or shrinks, leading to long overflow chains. Dynamic hashing provides a mechanism in which data buckets are added and removed dynamically and on-demand. 
*   **Extendible Hashing:** Uses a directory of pointers to buckets. It doubles the number of buckets by doubling the directory and splitting just the bucket that overflowed. 
*   **Linear Hashing:** Avoids the directory by using overflow pages and choosing buckets to split in a round-robin fashion.
