### Operations on Sequential Files

| Operation | Performance & Mechanism |
| :--- | :--- |
| **Search** | Because the file is sorted, the DBMS can use a binary search to find records much faster than a linear search. |
| **Insertion** | In a sorted file, when a new record arrives, it's not simply added to the end. Instead, we must locate the position where the record is to be inserted. If there is free space, insert there. If there is no free space, insert the record in an overflow block. In either case, the pointer chain must be updated. This ensures the final sequence remains sorted. |
| **Deletion** | Deletions use pointer chains to bridge the gap left by the removed record. |
| **Maintenance** | Adding or changing records often requires re-sorting the entire set, which can slow down your system. Therefore, you need to reorganize the file from time to time to restore sequential order. |
