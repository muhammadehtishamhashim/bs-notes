## Introduction to Advanced Data Models

In response to the increasing complexity of database applications (such as computer-aided design, computer-aided manufacturing, geographic information systems, and multimedia systems), two new data models have emerged: the Object-Oriented Data Model (OODM) and the Object-Relational Data Model (ORDM). This evolution represents third-generation DBMSs.

## Advantages Over Traditional Relational Model

Traditional relational DBMSs (RDBMSs) are incredibly successful for standard business applications but possess several weaknesses when handling complex applications. Advanced data models (Object-Oriented and Object-Relational) provide distinct advantages over these traditional relational weaknesses:

| Advantage | Description |
| :--- | :--- |
| **Better Representation of Real-World Entities** | The relational process of normalization generally leads to the creation of relations that do not correspond to "real-world" entities, causing fragmentation and unnatural joins. Advanced models provide enriched modeling capabilities that allow the real world to be modeled more accurately using complex objects and inheritance. |
| **Removal of Semantic Overloading** | The relational model uses only one construct—the relation—to represent both data and relationships, making it semantically overloaded. Advanced models explicitly define relationships and allow encapsulation of both state and behavior. |
| **Extensibility vs. Homogeneous Data Structures** | The relational model assumes horizontal and vertical homogeneity (every tuple must have the same attributes from the same domains). Advanced models allow new data types to be built from existing types (user-defined types) and allow attributes to hold complex, multi-valued data. |
| **More Expressive Operations** | The relational model has a fixed set of operations (like standard SQL). Advanced models support user-defined routines, methods, and navigational access from one object to the next, which is far more suitable for handling parts explosions and recursive queries. |
| **Removal of Impedance Mismatch** | In traditional systems, embedding declarative SQL into an imperative high-level language like C or Java produces an "impedance mismatch" because the models represent data differently. Advanced models (particularly OODBMSs) provide a single language interface, eliminating the inefficiencies of mapping between an application's object model and the database's relational model. |
