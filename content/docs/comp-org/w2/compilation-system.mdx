## The Compilation System

The `hello` program begins life as a high-level C program because it can be read and understood by human beings in that form. However, in order to run `hello.c` on the system, the individual C statements must be translated by other programs into a sequence of low-level *machine-language* instructions. These instructions are then packaged in a form called an *executable object program* and stored as a binary disk file. Object programs are also referred to as *executable object files*.

On a Unix system, the translation from source file to object file is performed by a compiler driver:

<div className="my-6 max-w-2xl bg-[#0d1117] dark:bg-black/50 border border-border/60 rounded-lg p-4 font-mono text-sm shadow-sm flex items-center gap-3">
  <span className="text-green-500 font-bold">linux&gt;</span> 
  <span className="text-foreground/90">gcc -o hello hello.c</span>
</div>

Here, the gcc compiler driver reads the source file `hello.c` and translates it into an executable object file `hello`. The translation is performed in the sequence of four phases. The programs that perform the four phases (*preprocessor, compiler, assembler*, and *linker*) are known collectively as the *compilation system*.

### The Four Phases of Translation

<div className="my-8 flex flex-col gap-4 relative z-0">
  
  {/* Preprocessor */}
  <div className="rounded-xl border border-l-4 border-l-orange-500 border-border/50 bg-card p-6 shadow-sm flex flex-col md:flex-row gap-6 items-start md:items-center">
    <div className="flex flex-col items-center justify-center bg-orange-500/10 border border-orange-500/20 rounded-lg w-24 h-24 shrink-0 shadow-sm relative overflow-hidden">
      <div className="absolute top-0 left-0 w-full h-1 bg-gradient-to-r from-orange-400 to-orange-600"></div>
      <span className="text-lg font-black text-orange-500 mb-1">.c → .i</span>
      <span className="text-xs font-bold font-mono text-orange-600/80 uppercase">cpp</span>
    </div>
    <div>
      <h4 className="text-xl font-bold mb-2 text-foreground flex items-center gap-2">
        <span className="bg-orange-500/10 text-orange-500 rounded-md px-2 py-0.5 text-sm font-bold border border-orange-500/20">1</span>
        Preprocessing Phase
      </h4>
      <p className="text-sm text-muted-foreground leading-relaxed">
        The preprocessor (`cpp`) modifies the original C program according to directives that begin with the `#` character. For example, the `#include <stdio.h>` command in line 1 of `hello.c` tells the preprocessor to read the contents of the system header file `stdio.h` and insert it directly into the program text. The result is another C program, typically with the `.i` suffix. In this step, it performs text-based modifications: it inserts header files, replaces macros (`#define`), and removes comments. 
      </p>
    </div>
  </div>

  {/* Compiler */}
  <div className="rounded-xl border border-l-4 border-l-blue-500 border-border/50 bg-card p-6 shadow-sm flex flex-col md:flex-row gap-6 items-start md:items-center">
    <div className="flex flex-col items-center justify-center bg-blue-500/10 border border-blue-500/20 rounded-lg w-24 h-24 shrink-0 shadow-sm relative overflow-hidden">
      <div className="absolute top-0 left-0 w-full h-1 bg-gradient-to-r from-blue-400 to-blue-600"></div>
      <span className="text-lg font-black text-blue-500 mb-1">.i → .s</span>
      <span className="text-xs font-bold font-mono text-blue-600/80 uppercase">cc1</span>
    </div>
    <div>
      <h4 className="text-xl font-bold mb-2 text-foreground flex items-center gap-2">
        <span className="bg-blue-500/10 text-blue-500 rounded-md px-2 py-0.5 text-sm font-bold border border-blue-500/20">2</span>
        Compilation Phase
      </h4>
      <p className="text-sm text-muted-foreground leading-relaxed">
        The compiler (`cc1`) translates the text file `hello.i` into the text file `hello.s`, which contains an *assembly-language program*. This phase converts high-level language into low-level language. The compiler performs several tasks: it checks for syntax errors, analyzes the program structure, optimizes the code, and translates C into assembly language. Assembly language is useful because it provides a common output language for different compilers for different high-level languages.
      </p>
    </div>
  </div>

  {/* Assembler */}
  <div className="rounded-xl border border-l-4 border-l-purple-500 border-border/50 bg-card p-6 shadow-sm flex flex-col md:flex-row gap-6 items-start md:items-center">
    <div className="flex flex-col items-center justify-center bg-purple-500/10 border border-purple-500/20 rounded-lg w-24 h-24 shrink-0 shadow-sm relative overflow-hidden">
      <div className="absolute top-0 left-0 w-full h-1 bg-gradient-to-r from-purple-400 to-purple-600"></div>
      <span className="text-lg font-black text-purple-500 mb-1">.s → .o</span>
      <span className="text-xs font-bold font-mono text-purple-600/80 uppercase">as</span>
    </div>
    <div>
      <h4 className="text-xl font-bold mb-2 text-foreground flex items-center gap-2">
        <span className="bg-purple-500/10 text-purple-500 rounded-md px-2 py-0.5 text-sm font-bold border border-purple-500/20">3</span>
        Assembly Phase
      </h4>
      <p className="text-sm text-muted-foreground leading-relaxed">
        Next, the assembler (`as`) translates `hello.s` into machine-language instructions (converting text instructions into binary instructions), packages them in a form known as a *relocatable object program*, and stores the result in the object file `hello.o`. This file is a binary file containing bytes to encode the instructions for function `main`. It provides the machine code, symbol information, and keeps track of unresolved references.
      </p>
    </div>
  </div>

  {/* Linker */}
  <div className="rounded-xl border border-l-4 border-l-green-500 border-border/50 bg-card p-6 shadow-sm flex flex-col md:flex-row gap-6 items-start md:items-center">
    <div className="flex flex-col items-center justify-center bg-green-500/10 border border-green-500/20 rounded-lg w-24 h-24 shrink-0 shadow-sm relative overflow-hidden">
      <div className="absolute top-0 left-0 w-full h-1 bg-gradient-to-r from-green-400 to-green-600"></div>
      <span className="text-lg font-black text-green-500 mb-1">.o → exe</span>
      <span className="text-xs font-bold font-mono text-green-600/80 uppercase">ld</span>
    </div>
    <div>
      <h4 className="text-xl font-bold mb-2 text-foreground flex items-center gap-2">
        <span className="bg-green-500/10 text-green-500 rounded-md px-2 py-0.5 text-sm font-bold border border-green-500/20">4</span>
        Linking Phase
      </h4>
      <p className="text-sm text-muted-foreground leading-relaxed">
        Notice that our `hello` program calls the `printf` function, which is part of the *standard C library* provided by every C compiler. The `printf` function resides in a separate precompiled object file called `printf.o`, which must somehow be merged with our `hello.o` program. The linker (`ld`) handles this merging. The result is the `hello` file, which is an executable object file (or simply *executable*) that is ready to be loaded into memory and executed by the system.
      </p>
    </div>
  </div>
</div>
