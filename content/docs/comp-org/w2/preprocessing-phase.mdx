# The Preprocessing Phase (`cpp`)

The preprocessor is the first program invoked by the compiler driver. It modifies the original C source code based on **preprocessor directives** (commands that start with `#`).

### Core Functions

The preprocessor performs purely **text-based operations**. It does not understand C syntax, types, or logic; it simply performs search-and-replace and text insertion.

1.  **File Inclusion (`#include`):** Copies the entire contents of the referenced header file (e.g., `<stdio.h>`) and pastes it directly into the source file.
2.  **Macro Expansion (`#define`):** Replaces macro names with their defined values or code snippets throughout the file.
3.  **Conditional Compilation (`#ifdef`, `#ifndef`):** Includes or excludes parts of the code based on certain conditions, useful for cross-platform development or debugging.
4.  **Comment Removal:** Strips out all `//` and `/* */` comments to reduce file size for the next phase.

## Comprehensive Summary
* **What it is:** The first phase of compilation focusing purely on text manipulation.
* **What it does:** Expands macros (`#define`), inserts header files (`#include`), processes conditional directives (`#ifdef`), and strips comments.
* **Why it matters:** It prepares a massive, complete, single text file (the translation unit) so the compiler has all the declarations and code it needs in one place without needing to search multiple files.
* **The Result:** A `.i` file that is much larger than the original `.c` file, containing only pure C code ready for actual syntax parsing.
