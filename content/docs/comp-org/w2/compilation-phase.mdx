# The Compilation Phase (`cc1`)

The compilation phase is the most complex and computationally heavy part of the compilation system. It takes the pure, expanded C code (`.i` file) and translates it into **Assembly Language** (`.s` file).

### The Internal Stages of the Compiler

The compiler doesn't just translate word-for-word. It deeply analyzes the code through several internal sub-phases:

1.  **Lexical Analysis (Scanning):** Breaks the stream of characters into meaningful tokens (keywords, identifiers, symbols).
2.  **Syntax Analysis (Parsing):** Checks if the tokens form valid C statements according to language grammar rules, creating an Abstract Syntax Tree (AST). *(This is where you get syntax errors like "missing semicolon".)*
3.  **Semantic Analysis:** Checks for logic and type consistency (e.g., trying to add a string to an integer).
4.  **Optimization:** Rewrites the code to make it faster or consume less memory (e.g., removing unused variables, evaluating constant math expressions at compile time).
5.  **Code Generation:** Converts the optimized abstract representation into target-specific assembly instructions.

## Comprehensive Summary
* **What it is:** The brain of the translation system, converting C constructs into low-level CPU instructions.
* **What it does:** Performs lexical, syntax, and semantic checks. If errors exist, compilation halts here. It then optimizes the code and translates it into hardware-specific Assembly language.
* **Why it matters:** It abstracts away the hardware. Programmers write high-level logic, and the compiler handles the complex task of figuring out which registers and CPU instructions to use.
* **The Result:** A `.s` file containing text-based assembly instructions (mnemonics like `mov`, `push`, `add`) specific to the target architecture.
