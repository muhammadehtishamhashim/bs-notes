# The Linking Phase (`ld`)

The linker is the final phase of the compilation system. Real-world programs are rarely built from a single file. They consist of multiple `.c` files compiled into multiple `.o` files, plus pre-compiled system libraries (like the Standard C Library). The linker's job is to glue them all together.

### The Two Main Tasks of the Linker

1.  **Symbol Resolution:** * Our `hello.o` file has an unresolved reference to `printf`. 
    * The linker searches through provided object files and system libraries (e.g., `libc.so` or `printf.o`) to find the exact definition of `printf`. If it can't find it, it throws a **Linker Error** (e.g., `undefined reference to 'main'`).
2.  **Relocation:**
    * Object files are assembled starting at address 0.
    * The linker merges all the code sections into a single large file and assigns **absolute memory addresses** to every function and global variable.
    * It then updates all the placeholder references in the code to point to these new, correct absolute addresses.

## Comprehensive Summary
* **What it is:** The final glue that binds separate modular pieces of code into a single working program.
* **What it does:** Resolves missing symbols (functions/variables) by looking through other object files and system libraries. It then relocates memory addresses so the program can be loaded into RAM sequentially.
* **Why it matters:** It allows modular programming. You don't have to compile the massive C Standard Library every time you write a program; the linker just links your tiny `hello.o` to the pre-compiled library.
* **The Result:** A final, complete binary executable file. The compilation process is fully complete, and typing `./hello` in the terminal will tell the OS loader to place this file in memory and execute it.
