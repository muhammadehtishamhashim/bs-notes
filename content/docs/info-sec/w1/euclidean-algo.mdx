# Euclidean Theory and Algorithm

When dealing with massive numbers in cryptography, listing factors or finding prime trees is computationally impossible. The **Euclidean Algorithm** provides a lightning-fast mathematical approach to finding the Greatest Common Divisor (GCD).

<Callout title="Prerequisite Rule" type="warn">
Always arrange your numbers such that $A > B$ before starting the algorithm.
</Callout>

---

## The Tabular Method (Q, A, B, R)

The algorithm repeatedly divides $A$ by $B$ to find a Quotient ($Q$) and a Remainder ($R$). 
For the next row, $A$ inherits the old $B$, and $B$ inherits the old $R$. This continues until the Remainder ($R$) becomes $0$. The last non-zero $B$ is your GCD.

## The Programmatic Algorithm

In programming languages, the Euclidean algorithm is incredibly easy to write using recursion and the modulo (`%`) operator.

The logic is defined mathematically as:
$$ \text{GCD}(a, b) = \text{GCD}(b, a \pmod b) $$

**Pseudocode Structure:**
```text
Function Euclid_GCD(a, b):
    If b == 0, then:
        return a
    Else:
        return Euclid_GCD(b, a % b)
```

### Algorithm Tracing Examples

**Example 1: Find GCD(50, 12)**
1. `GCD(50, 12)` $\implies$ `GCD(12, 50 % 12)` $\implies$ `GCD(12, 2)`
2. `GCD(12, 2)` $\implies$ `GCD(2, 12 % 2)` $\implies$ `GCD(2, 0)`
3. `GCD(2, 0)` $\implies$ Since $b=0$, return $a$, which is **$2$**.
* **Result:** **$2$**

**Example 2: Find GCD(83, 90)** *(Remember: $a$ must be $> b$, so flip them)*
1. `GCD(90, 83)` $\implies$ `GCD(83, 90 % 83)` $\implies$ `GCD(83, 7)`
2. `GCD(83, 7)` $\implies$ `GCD(7, 83 % 7)` $\implies$ `GCD(7, 6)`
3. `GCD(7, 6)` $\implies$ `GCD(6, 7 % 6)` $\implies$ `GCD(6, 1)`
4. `GCD(6, 1)` $\implies$ `GCD(1, 6 % 1)` $\implies$ `GCD(1, 0)`
5. `GCD(1, 0)` $\implies$ Since $b=0$, return $a$, which is **$1$**.
* **Result:** **$1$**
