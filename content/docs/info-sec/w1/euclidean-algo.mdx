# Euclidean Theory and Algorithm

When dealing with massive numbers in cryptography, listing factors or finding prime trees is computationally impossible. The **Euclidean Algorithm** provides a lightning-fast mathematical approach to finding the Greatest Common Divisor (GCD).

<Callout title="Prerequisite Rule" type="warn">
Always arrange your numbers such that $A > B$ before starting the algorithm.
</Callout>

---

## The Tabular Method (Q, A, B, R)

The algorithm repeatedly divides $A$ by $B$ to find a Quotient ($Q$) and a Remainder ($R$). 
For the next row, $A$ inherits the old $B$, and $B$ inherits the old $R$. This continues until the Remainder ($R$) becomes $0$. The last non-zero $B$ is your GCD.

## The Programmatic Algorithm

In programming languages, the Euclidean algorithm is incredibly easy to write using recursion and the modulo (`%`) operator.

The logic is defined mathematically as:
$$ \text{GCD}(a, b) = \text{GCD}(b, a \pmod b) $$

**Pseudocode Structure:**
```text
Function Euclid_GCD(a, b):
    If b == 0, then:
        return a
    Else:
        return Euclid_GCD(b, a % b)
```

### Algorithm Tracing Examples

**Example 1: Find GCD(50, 12)**
1. `GCD(50, 12)` $\implies$ `GCD(12, 50 % 12)` $\implies$ `GCD(12, 2)`
2. `GCD(12, 2)` $\implies$ `GCD(2, 12 % 2)` $\implies$ `GCD(2, 0)`
3. `GCD(2, 0)` $\implies$ Since $b=0$, return $a$, which is **$2$**.
* **Result:** **$2$**

## Exam Critical Points
<div className="bg-red-500/5 border-l-4 border-red-500 p-4 rounded-r-xl my-6">
  <ul className="list-disc pl-5 space-y-2 text-sm font-medium text-foreground">
    <li><strong className="text-red-500">Termination Condition:</strong> The algorithm strictly stops when the remainder ($R$) becomes $0$.</li>
    <li><strong className="text-red-500">The Result:</strong> The GCD is always the <strong>last non-zero remainder</strong> (or the value of $B$ right before $R$ becomes $0$).</li>
    <li><strong>Programmatic Rule:</strong> The recursive structure is $\text{GCD}(A, B) \rightarrow \text{GCD}(B, A \pmod B)$.</li>
    <li>Ensure $A \geq B$ before starting. If $B > A$, the first algorithmic step naturally swaps them (e.g., $12 \pmod{50} = 12$, causing the next step to be $\text{GCD}(50, 12)$).</li>
  </ul>
</div>

## Comprehensive Summary
* **What it is:** A recursive division algorithm based on Euclid's Division Lemma ($A = BQ + R$).
* **Core Mechanism:** It continuously shifts the divisor ($B$) into the dividend position ($A$), and the remainder ($R$) into the divisor position ($B$) until the remainder is zero. 
* **Why it matters:** Finding prime factors of 1000-digit numbers is physically impossible for modern computers. The Euclidean algorithm skips factorization entirely, finding the GCD using simple, fast division.
